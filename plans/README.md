# Plans — Технические планы

Место для планов реализации функций. Один план = одна функция.

## Правила

1. **Один план = одна функция.** Если план уже существует — работаем с ним, не создаём новый.

2. **Имя файла:** `YYYY-MM-DD-название-функции.md` — текущий год, месяц, день, название функции латиницей.

3. **Структура плана:**
   - Заголовок с названием функции
   - Описание функции (1-2 предложения)
   - Фазы со статусами: `[ ]` (в работе) или `[x]` (выполнено)
   - Итоговый блок в конце: реализовано целиком/нет, что осталось

4. **Актуализация:** любой агент, работающий с планом, обязан обновлять его после каждой сессии.

## Цикл работы

- **Первый чат:** формирует план функции.
- **Второй чат:** разбивает на фазы.
- **Третий чат:** проверяет каждую фазу на адекватность.
- **После проверки:** реализация.

## Пример

```
# 2026-07-04-audit-online-funnel.md

## Фазы

- [ ] Фаза 1. Анализ текущей воронки
- [ ] Фаза 2. Поиск точек оттока
- [ ] Фаза 3. Рекомендации по улучшению

## Итог

[Заполняется по мере выполнения]
```
