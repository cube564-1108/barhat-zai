# Triggers — автоматизации

> Триггеры — это агенты которые работают по расписанию или по условию.
> «Триггер — это твоя мысль, которую AI делает регулярно вместо тебя.» — Миллер

## Типы триггеров

| Тип | Как запускается | Примеры |
|---|---|---|
| Cron / расписание | По времени (ежедневно, еженедельно) | Утренний дайджест, еженедельный отчёт |
| По событию | При наступлении условия | Брошенная корзина, низкий остаток |
| По запросу | Вручную, по команде | Аналитика за период, экспорт |

## Создать новый триггер

```
Промпт:
"Создай триггер который [что делает].
Запускается [как часто / по какому условию].
Источник данных: [API / БД / файл].
Результат отправляет: [в Telegram / в файл / email]."
```

После создания кода:
1. Разместить в `triggers/[имя-триггера].py` (или нужный язык)
2. Задокументировать в этом файле (таблица ниже)
3. Задеплоить на Amvera как worker-процесс

## Реестр триггеров

| Имя | Что делает | Расписание | Статус |
|---|---|---|---|
| утренний-дайджест | Метрики за день → Telegram | 08:00 ежедневно | ⏳ создать |

## Архитектура воркера (Python + Amvera)

```python
# triggers/trigger-name.py
import schedule
import time

def run():
    # 1. Получить данные из API
    # 2. Обработать
    # 3. Отправить результат

schedule.every().day.at("08:00").do(run)

while True:
    schedule.run_pending()
    time.sleep(60)
```

## Деплой на Amvera

```bash
# Добавить в Procfile или amvera.yml:
worker: python triggers/trigger-name.py
```
