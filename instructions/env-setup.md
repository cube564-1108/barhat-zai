# Инструкция: Создание .env файла для API-ключей

## Что это и зачем

Файл `.env` хранит секретные данные (API-ключи, пароли) которые **НЕ должны** попасть в git. Это безопасный способ работать с чувствительными данными.

## Шаг 1: Скопировать шаблон

### В Windows (PowerShell):
```powershell
Copy-Item .env.example .env
```

### В Windows (cmd):
```cmd
copy .env.example .env
```

### В Git Bash / Linux:
```bash
cp .env.example .env
```

### Вручную:
1. Открой файл `.env.example` в редакторе
2. Файл → Сохранить как...
3. Имя файла: `.env`
4. Сохранить в корне проекта (рядом с `.env.example`)

---

## Шаг 2: Получить RetailCRM API-ключ

1. Открой https://barhatretailcrm.retailcrm.ru
2. Авторизуйся
3. Меню: **Настройки** → **Ключи доступа к API**
4. Нажми кнопку **Добавить**
5. Заполни форму:
   - **Комментарий:** `Сверка с МойСклад`
   - **Тип доступа:** `Доступ ко всем магазинам` (или выбери нужный)
   - **Разрешенные действия:**
     - Заказы: получение ✅
     - (остальное можно выключить)
6. Нажми **Сохранить**
7. **Скопируй ключ** — он показывается только один раз!
   - Выглядит как: `snWb...ck7H` (первые 4 и последние 4 символа)

**Вставь ключ в `.env`:**
```
RETAILCRM_API_KEY=скопированный_ключ
```

---

## Шаг 3: Получить доступ к МойСклад

### Вариант А — Basic Auth (самый простой, рекомендуется):

**НЕ нужно ничего создавать в интерфейсе!** Просто используй свой логин и пароль от МойСклад.

**Вставь в `.env`:**
```
MOYSKLAD_LOGIN=твой_email_или_логин
MOYSKLAD_PASSWORD=твой_пароль
```

Это работает сразу — скрипт сам сформирует Basic Auth заголовок.

---

### Вариант Б — Токен доступа (если Basic Auth не работает):

Если Basic Auth по какой-то причине не сработает, можно создать токен.

1. Открой https://online.moysklad.ru и авторизуйся
2. Меню: **Настройки** → **Токены**
3. Нажми **Добавить токен**
4. Заполни:
   - **Название:** `Сверка с RetailCRM`
   - Отметь нужные права (или все)
5. Нажми **Сохранить**
6. **Скопируй токен** — показывается только один раз!

**Вставь токен в `.env`:**
```
MOYSKLAD_TOKEN=скопированный_токен
```

---

### Что выбрать?

**Начни с Варианта А (Basic Auth):**
- Проще — ничего не нужно искать в интерфейсе
- Надежно — работает сразу
- Если не сработает — перейдем на токен

**Вставь в `.env`:**
```
MOYSKLAD_LOGIN=твой_логин_от_мойсклад
MOYSKLAD_PASSWORD=твой_пароль_от_мойсклад
```

## Шаг 4: Проверить .env

Файл `.env` должен выглядеть примерно так:

```env
# RetailCRM
RETAILCRM_API_URL=https://barhatretailcrm.retailcrm.ru
RETAILCRM_API_KEY=snWb4Xz8ck7H

# МойСклад
MOYSKLAD_API_URL=https://api.moysklad.ru/api/remap/1.2
MOYSKLAD_TOKEN=eyJ0eXAiOiJKV1QiLCJhbGc...

# Или Basic Auth:
# MOYSKLAD_LOGIN=user@example.com
# MOYSKLAD_PASSWORD=secret123
```

---

## Шаг 5: Проверить .gitignore

Убедись, что `.env` **НЕ попадет в git**. Проверь файл `.gitignore` в корне:

```bash
# В Git Bash или терминале:
cat .gitignore | grep .env
```

Должно быть видно:
```
.env
```

Если нет — добавь строку `.env` в `.gitignore`.

---

## Безопасность

- ❌ **НЕ коммить** `.env` в git
- ❌ **НЕ отправлять** `.env` в чат/почту
- ❌ **НЕ делиться** ключами с посторонними
- ✅ `.env.example` — это нормально (он без реальных ключей)
