# Лендинг отчетов Бархат

Веб-интерфейс для формирования отчетов по сверке заказов между RetailCRM и МойСклад.

## Установка

```bash
cd reports
pip install -r requirements.txt
```

## Запуск

```bash
python app.py
```

Сервер запустится на `http://localhost:5000`

## Использование

1. Открой в браузере: `http://localhost:5000`
2. Выбери вкладку "Сверка заказов"
3. Укажи период для отчета
4. Нажми "Сформировать отчет"
5. Дождись загрузки данных (это может занять время для больших периодов)
6. Изучай результаты сверки

## Что показывает отчет

### Статистика

- 📊 **Всего в CRM** — количество заказов за период
- 📊 **Всего в МойСклад** — количество заказов за период
- ❌ **Не найдены** — заказы из CRM, которых нет в МойСклад
- ⚠️ **С расхождениями** — заказы с разными статусами или суммами
- ✅ **Совпадают** — заказы полностью совпадают

### Детализация

#### Не найденные заказы

Заказы из CRM со статусами "выполнен", "отменен" или "удержание", но отсутствующие в МойСклад. Это критические расхождения.

#### Расхождения

Заказы, которые есть в обеих системах, но с отличиями:
- **Статус** — разные статусы в CRM и МойСклад
- **Сумма** — разные суммы (погрешность более 1 рубля)

## Логика сверки

### Статусы для синхронизации

В МойСклад подгружаются только заказы со статусами:
- ✅ выполнен
- ❌ отменен
- ⏸ удержание

Заказы с другими статусами в CRM сверяться с МойСклад не должны.

### Поиск соответствия

Заказы ищутся по трём критериям:
1. `externalCode` в МойСклад = ID заказа в CRM
2. `description` в МойСклад содержит ID заказа CRM
3. Номер заказа совпадает

### Допустимые отклонения

- Сумма: погрешность до 1 рубля
- Статус: нормализованные названия (completed/выполнен считаются одинаковыми)

## API

### POST /api/reconcile

Запустить сверку за период.

**Body:**
```json
{
  "from": "2026-07-01T00:00",
  "to": "2026-07-07T23:59"
}
```

**Response:**
```json
{
  "summary": {
    "total_crm": 1500,
    "total_ms": 800,
    "not_found_in_ms": 50,
    "with_differences": 25,
    "fully_match": 725
  },
  "not_found_in_ms": [...],
  "with_differences": [...],
  "fully_match": [...]
}
```

### GET /api/health

Проверка состояния сервера и доступности сервисов.

**Response:**
```json
{
  "status": "ok",
  "timestamp": "2026-07-07T12:00:00",
  "services": {
    "retailcrm": true,
    "moysklad": true
  }
}
```

## Troubleshooting

### Ошибка "Сервис недоступен"

Проверь, что `.env` файл находится в корне проекта и содержит:
```
RETAILCRM_API_URL=https://barhatretailcrm.retailcrm.ru
RETAILCRM_API_KEY=твой_ключ
MOYSKLAD_API_URL=https://api.moysklad.ru/api/remap/1.2
MOYSKLAD_LOGIN=твой_логин
MOYSKLAD_PASSWORD=твой_пароль
```

### Медленная загрузка

Для больших периодов (месяц и более) выгрузка может занимать несколько минут. Это нормально — API имеет ограничения по скорости.

### Ошибка 415 от МойСklad

Эта ошибка уже обработана в скриптах. Если всё равно появляется — проверь версию requests: `pip install --upgrade requests`
