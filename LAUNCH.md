### День 1 — Контекст проекта (2–3 часа)

**Шаг 1.1 — Заполни CLAUDE.md**

Открой `CLAUDE.md`. Замени все `[PLACEHOLDER]` на реальные данные:
- 2-3 предложения что за проект и для кого
- Стек (язык, фреймворк, хостинг)
- Структура файлов (появится по мере создания)

> Лимит: 200 строк. Это трудовой договор агента — он читается каждую сессию.

**Шаг 1.2 — Заполни .business/wiki/index.md**

Открой `.business/wiki/index.md` и заполни:
- Суть проекта (1 абзац)
- Для кого
- Как зарабатывает
- Текущий приоритет №1

Детальные файлы в подпапках заполнишь позже. Wiki — сжатая карта для агента.

**Шаг 1.3 — Создай GitHub репозиторий**

```
Промпт для Claude:
"Создай приватный репозиторий на GitHub с названием [название].
Добавь все файлы кроме .env и .business/.
Сделай первый commit с сообщением 'init: project structure'."
```

### День 2 — ai-clone (голосовое интервью, ~45 мин)

ai-clone — цифровая проекция тебя. Агент читает его и действует от твоего имени.

Проводи интервью по одному блоку за раз. Промпт для каждого блока:

```
"Ты — интервьюер. Собираешь мой профиль для AI-Clone.
Тема блока: [identity / voice / thinking / principles / style].
Задавай вопросы по одному. После ответа — следующий вопрос.
В конце блока — запиши всё в соответствующий файл ai-clone/[папка]/[файл].md"
```

**Порядок блоков:**
1. `identity/` — кто ты, ценности, биография (15 мин)
2. `voice/` — как ты говоришь, стоп-слова (10 мин)
3. `thinking/` — как принимаешь решения (10 мин)
4. `principles/` — что важно в продукте и бизнесе (10 мин)

Блок `feedback/` — не заполнять заранее. Только из реальных ошибок в работе.

### День 3 — mastery (методологии)

Добавь 1 область с 3-5 авторами, актуальными для твоего проекта.

**Структура:**
```
mastery/
└── [область]/          ← например: marketing, sales, operations
    ├── INDEX.md
    └── [автор].md      ← кто + 1-3 метода с шагами «что→как→когда»
```

**Промпт:**
```
"Я запускаю проект в области [ниша].
Найди 3-5 авторов, чьи методологии доказанно работают в этой нише.
По каждому: кто + 1-3 лучших метода с пошаговым алгоритмом.
Формат: mastery/[область]/[автор].md"
```

Miller уже в `mastery/vibe-coding/miller.md` — это всегда база.

> **Сканируй делателей, не говорунов (Поток 4):** для практических навыков бери не советы
> «как надо», а реальный контент автора (20 роликов / все посты) → «выдели повторяющиеся паттерны».
>
> **Универсальное мастерство `idea-validation`:** создай его первым, с триггер-словом
> «испытать идею» — гейт, который проверяет любую новую фичу на адекватность ДО плана.

### День 4 — .business/ (второй мозг)

Заполни 5 ключевых разделов. По одному за сессию, через интервью с Claude:

```
"Собери .business/[раздел]/ для проекта [название].
Задавай вопросы по одному. Записывай ответы в файлы.
В конце добавь ссылки между связанными файлами [[файл]]."
```

**Приоритет разделов:**
1. `audience/` — аватар клиента, боли, желания
2. `products/` — что продаём, почём
3. `economics/` — юнит-экономика, точка безубыточности
4. `goals/` — цель на год, квартал, KPI
5. `marketing/` — каналы, конкуренты, контент

### День 5 — Первый план и первая фича

Теперь у тебя есть контекст. Время строить.

**Выбери первую задачу** — ту, что даст результат быстрее всего.

**Цикл трёх чатов (обязательно):**
```
Чат 1: "Создай план реализации [задача]. Найди как другие решали."
        → сохрани в plans/YYYY-MM-DD-название.md

Чат 2: "Прочитай план. Разбей на фазы с чекбоксами [ ] / [x]."
        → обновить план

Чат 3: "Проверь каждую фазу по 4 вопросам:
        1. Решает ли это проблему?
        2. Эффективное ли это решение?
        3. Не код ли это ради кода?
        4. Не усложнение ли для MVP?
        Затем ДОКАЖИ, что этот план — лучшее решение.
        И pre-mortem: 'прошёл год, система провалилась — почему?'"
        → финальный план

Чат 4+: реализация фаза за фазой
```

> **Метод «докажи» (Поток 4):** ИИ зарабатывает на токенах → пишет «код ради кода».
> На каждом шаге заставляй доказать эффективность — второй проход даёт реальное решение.

## Когда добавлять новый урок Миллера

1. Создать конспект: `mastery/miller-archive/уроки/урок-N-поток-M.md`
2. Транскрипт: `mastery/miller-archive/уроки/транскрипт-урок-N-поток-M.md`
3. Обновить индекс: `mastery/miller-archive/index.md` (таблица статусов)
4. Новые принципы → в тематические файлы `mastery/miller-archive/*.md`
5. Ключевые методы → обновить `mastery/vibe-coding/miller.md`
