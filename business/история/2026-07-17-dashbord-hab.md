# Главный дашборд-хаб

## Задача
Создать главную страницу-дашборд (хаб), откуда будут доступны все отчеты и проекты компании.

## Как решал

### Шаг 1: Определение требований
- Навигация: sidebar слева с категориями
- Загрузка: iframe для изоляции отчетов
- Конфигурация: JSON-файл для управления ссылками
- Дизайн: фирменный стиль Бархат

### Шаг 2: Первая реализация (не сработало)
- Создал `index.html` с sidebar и iframe
- Создал `dashboard-config.json` с конфигом
- Использовал внешние файлы `/brand/tokens.css` и `/brand/brand.css`

**Проблема:** при локальном открытии стили не применились — всё было белым/черным

### Шаг 3: Исправление стилей
- Встроил CSS переменные напрямую в `index.html`
- Добавил Google Fonts для Vollkorn и Inter
- Стили применились, но...

### Шаг 4: Исправление загрузки конфига
- Браузер блокировал `fetch('/dashboard-config.json')` при локальном открытии (CORS)
- Показывалось "Ошибка загрузки конфигурации"

**Решение:** встроил конфиг прямо в JavaScript как объект `const config = {...}`

### Шаг 5: Финальная настройка путей
- Изменил URL отчета с `/florist-quality-dashboard.html` на `./florist-quality-dashboard.html`
- Всё заработало

## Результат
**Да** — главный дашборд создан и работает локально.

## Что можно было лучше
1. Сразу встроить конфиг в HTML, а не через fetch
2. Использовать относительные пути с самого начала
3. Предусмотреть fallback для серверной работы

## Было / Стало

**Было:**
- Текущий отчет (`index.html`) — конкретный отчет по качеству
- Нет единой точки входа
- Каждый отчет открывается отдельно

**Стало:**
- [index.html](index.html) — главный дашборд-хаб
- Sidebar слева с категориями
- Отчеты загружаются в iframe
- Фирменный стиль Бархат (винный фон, розовые акценты, Vollkorn + Inter)
- Простое добавление отчетов через редактирование конфига в HTML

## Файлы
- `index.html` — главный дашборд
- `florist-quality-dashboard.html` — отчет по качеству (остался на месте)
- `dashboard-config.json` — больше не используется (конфиг в HTML)
