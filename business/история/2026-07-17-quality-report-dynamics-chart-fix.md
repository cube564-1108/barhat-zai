# Сессия: Исправление кнопки Динамика и улучшение графика

**Дата:** 2026-07-17

## Задача

Исправить неработающую кнопку "Динамика" в отчёте по качеству (quality-report-14-18.html) и улучшить отображение графика.

## Что сделано

### 1. Исправление кнопки Динамика

**Проблема:** Кнопка "📈 Динамика" не работала из-за неправильного escaping апострофа в onclick-атрибуте.

**Решение:** Перешёл с передачи JSON через onclick на хранение данных в data-атрибутах:
- `data-salon` — название салона (с `&apos;` для апострофов)
- `data-periods` — JSON с данными периодов
- Новая функция `showTrendFromBtn(btn)` читает атрибуты и вызывает `showTrend()`

**Файл:** `scripts/generate_quality_report_14_18.py` (строки 765-866)

### 2. Изменение данных графика с процентов на баллы

**Было:** График показывал проценты (0-100%)
**Стало:** График показывает средний балл (8-20)

**Изменения:**
- `pData['14'].percent` → `pData['14'].avg`
- `pData['18'].percent` → `pData['18'].avg`
- Оси Y: min=8, max=18 (потом 20)
- Tooltip: "14 баллов: 13.5/14" вместо "95%"

### 3. Хронологическая сортировка дат

**Проблема:** Даты сортировались лексикографически ("10.2025" < "2.2025")

**Решение:** Добавлена функция `periodValue()` для правильной сортировки по ГГГГММ:
```javascript
const periodValue = function(periodStr) {
    const parts = periodStr.split('.');
    return parseInt(parts[1]) * 100 + parseInt(parts[0]); // ГГГГММ
};
const sortedPeriods = Object.keys(periodsData).sort((a, b) => periodValue(a) - periodValue(b));
```

### 4. Форматирование дат

**Было:** "7.2026", "6.2026"
**Стало:** "июл.26", "июн.26"

Добавлена функция `formatPeriod()` для конвертации periods в читаемый формат.

### 5. Подписи данных на графике

Добавлен плагин `datalabels` для отображения значений над каждой точкой графика (через `afterDatasetsDraw`).

## Результат

**Статус:** ✅ Выполнено

- Даты идут последовательно слева направо
