# Рефлексия: Аналитика по продажам — планирование и начало реализации

**Дата:** 2026-07-17
**Задача:** Создать раздел «Аналитика → Продажи» в главном дашборде
**Статус:** Планирование завершено, Фаза 0 реализована

---

## 1. Задача

Создать аналитику по продажам в дашборде Бархат:
- Сумма отгрузки за текущий месяц в разрезе по салонам
- Сравнение периода с прошлым годом
- Статистика по закрытым месяцам текущего года vs прошлый год

**Источник данных:** RetailCRM API

---

- **Чат 4:** Реализация фаза за фазой

### Чат 3: Критика
Провёл критику каждой фазы. Выявил риски:
- Хардкод салонов и статусов → **Решение:** конфиг файлы
- Нет кэширования → **Решение:** кэш в JSON с TTL
- Нет loading states → **Решение:** spinner + skeleton
- CDN зависимость → **Решение:** локальная копия Chart.js

Предложил 3 варианта: A (быстро), B (баланс), C (максимум)
**Выбор владельца:** Вариант C — максимум качества
