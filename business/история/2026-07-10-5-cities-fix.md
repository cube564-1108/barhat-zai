# Исправление: почему было 4 города вместо 5

**Дата:** 2026-07-10
**Проблема:** В аналитике отображались только 4 города из 5 (не хватало Челябинска)

## Диагностика

### Проверка формы Pyrus
В форме "Оценка качества сборки букетов" (#1327961) есть **12 салонов**:

| Город | Салоны |
|-------|--------|
| НСК | БХ, ЖД, Левый, Восход |
| ЕКБ | Бажова, Белинского |
| Томск | ДК, Фрунзе |
| БРН | Совесткая, Лазурная |
| **Челябинск** | **Цвиллинга, Свердловскй** |

### Первичный анализ данных
Выгрузка без параметров: **5,459 задач** — Челябинск отсутствовал

## Причина проблемы

**API Pyrus по умолчанию возвращает только активные (незавершённые) задачи.**

Завершённые и заархивированные задачи не включаются в выгрузку без специального параметра.

## Решение

### 1. Добавлен параметр `include_archived=true`

**Файлы обновлены:**
- `scripts/generate_dashboard_from_pyrus.py`
- `pyrus_export.py`
- `scripts/generate_dashboard_from_json.py` (новый)

```python
params['include_archived'] = 'true'
```

### 2. Создан скрипт для генерации из JSON

[`scripts/generate_dashboard_from_json.py`](scripts/generate_dashboard_from_json.py) — генерация дашборда из локального JSON файла Pyrus

### 3. Исправлен маппинг БРН

**Было:** `'БРН': 'Барнаул'` (неправильно!)  
**Стало:** `'БРН': 'Брянск'` (правильно)

## Результат

```bash

# 2. Генерация дашборда из JSON

# 3. Открыть дашборд

```
