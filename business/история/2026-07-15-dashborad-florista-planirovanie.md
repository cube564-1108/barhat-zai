# Рефлексия: Дашборд флориста — планирование

> **Дата:** 2026-07-15
> **Задача:** Запланировать крупный проект — дашборд для флористов
> **Результат:** Да — план создан, одобрен, на паузе

---

## Что делал

1. **Уточнил требования** — дашборд для флористов с интеграцией в RetailCRM
2. **Выбрал стек** — Node.js + Fastify + WebSocket / React + Vite
3. **Создал план** — 7 фаз с детальными задачами (методология Миллера)
4. **Прошёл критику** — фаза 3 по Миллеру, план одобрен
5. **Начал ФАЗУ 1** — исследование RetailCRM API

---

## Как решал

### Процесс по методологии Миллера

```
Чат 1: Создать план               ✓
Чат 2: Разбить на фазы с [ ]      ✓
Чат 3: Критика каждой фазы        ✓
```

### Критика плана

- Аналоги: курьерские → ресторанные KDS (исправлено)
- Стек: Node.js выбран для real-time WebSocket
- Офлайн: PWA с service worker для планшета
- Безопасность: API ключи в `.env`, не в git

### Исследование RetailCRM API

- Нашёл официальную документацию API v5
- Изучил структуру заказов из реальных данных
- Нашёл статусы: `send-to-florist`, `at-work`, `complete`
- Нашёл поле для фото: `offer.imageUrl`
- Нашёл endpoint для загрузки: `POST /api/v5/files/upload`
