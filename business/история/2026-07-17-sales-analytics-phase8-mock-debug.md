# Сессия: Аналитика продаж — Фаза 8 (Mock режим)

## Задача

Отладить ошибку HTTP 500 при загрузке аналитики продаж на Amvera.
SSL ошибка при подключении к RetailCRM API.

## Как решал

### Попытка 1: SSL фиксы с urllib3
- Добавлены различные SSL контексты
- `OP_LEGACY_SERVER_CONNECT`, `set_ciphers('DEFAULT:@SECLEVEL=0')`
- Логирование перенесено в stderr
- **Результат:** Не помогло, ошибка 500 сохраняется

### Попытка 2: Fallback на requests
- Добавлен `requests` библиотека как альтернатива urllib3
- `requests.get(verify=False)` для проблемных SSL
- Логирование версий Python/urllib3
- **Результат:** Ещё не протестировано на Amvera

### Попытка 3: Mock режим для тестирования фронтенда
- Добавлен `SALES_MOCK_MODE=true` в .env и app.py
- Созданы mock данные с реальной структурой API
- **Результат:** Фронтенд заработал, но найдены проблемы:

### Исправление mock данных и фронтенда
1. **Названия салонов:** Исправлены на реальные из salons.json (13 салонов)
2. **Фильтры периодов:** Добавлена реальная логика в `onPeriodChange()`, `applyCustomPeriod()`
3. **Месячный график:** Добавлено сравнение с прошлым годом (2 dataset)
4. **Фикс бага:** Исправлен `await` в `loadPeriodData()`
