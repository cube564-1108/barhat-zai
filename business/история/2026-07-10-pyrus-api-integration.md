# 2026-07-10 — Интеграция с Pyrus API

## Задача
Настроить выгрузку данных из формы Pyrus "Анкета для бархат зай" по API.

## Как решал

### 1. Документация
Создал подробную инструкцию по работе с Pyrus API в `.business/pyrus-api-instruction.md`

### 2. Аутентификация
- Получил токен от пользователя
- Обнаружил что Pyrus использует нестандартную auth: POST /v4/auth с `login` + `security_key`
- В ответ приходит JWT access_token для дальнейших запросов

### 3. Структура API
- **Форма:** `/v4/forms/{id}` — структура полей
- **Регистр (заявки):** `/v4/forms/{id}/register` — данные заявок (не `registers`!)
- Пагинация через `next_page_token`

### 4. Экспорт данных
- Создал скрипт `pyrus_export.py` для выгрузки
- Получил 5,459 заявок из формы ID 1327961
- Экспортировал в CSV (4.5 MB) и JSON (34 MB)

### 5. Структура данных
Заявки содержат массив `fields` (не `values` как ожидалось):
- `multiple_choice`: `{choice_names: [...]}`
- `text/plain`: прямое значение
- `file`: `{id, name}`

## Результат
**Да.** 5,459 заявок успешно выгружены в `pyrus_export_final.csv`

## Что можно было лучше
1. Сразу проверить правильность токена и метода auth в документации Pyrus
2. Использовать правильные кодировки для Windows-терминала с первого раза
