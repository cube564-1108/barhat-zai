# Рефлексия: Исправление task_id в дашборде флористов

**Дата:** 2026-07-16
**Задача:** Исправить пустой блок "Задачи с низким баллом" в дашборде флористов
**Результат:** Да — проблема найдена и исправлена

---

## 1. Задача

Блок "Задачи с низким баллом" в дашборде florist-quality-dashboard.html не отображал задачи.

---

## 2. Как решал

### Поиск проблемы

1. **Проверил JavaScript код** — функция `updateProblemTasks()` требует `task_id`:
   ```javascript
   if (!order.salon || !order.total_score || !order.task_id) return;
   ```

2. **Проверил данные в HTML** — обнаружил, что у всех заказов пустой `task_id`:
   ```
   Заказов с task_id: 0/14,980
   ```

3. **Нашёл причину** — в цепочке передачи данных терялось поле `task_id`:
   - Pyrus API → `task.id` есть ✓
   - `pyrus_to_csv_format()` → сохраняет в `record['task_id']` ✓
   - `generate_dashboard()` → **НЕ передаёт** `task_id` ✗
   - `prepare_data_for_js()` → не получает `task_id` ✗

### Решение

Добавил `'task_id': record.get('task_id', '')` в `scripts/generate_dashboard_from_pyrus.py`:

```python
data.append({
    'task_id': record.get('task_id', ''),  # ← ДОБАВИЛ
    'city': '',
    'period': record['период'],
    ...
})
```

---

## 3. Результат

**Да.** Проблема исправлена.

**Что исправлено:**
- В `scripts/generate_dashboard_from_pyrus.py` добавлено поле `task_id`
- HTML перегенерирован из Pyrus API
- Теперь все 14,990 заказов имеют `task_id`
- Блок "Задачи с низким баллом" будет отображать 8,892 проблемных задач

**Верификация:**
```
Заказов с task_id: 14,990/14,990 ✓
Проблемных задач: 8,892 ✓
```

---

## 4. Что можно было лучше

1. **Тестирование после каждого изменения** — если бы запускались тесты после изменения `generate_dashboard()`, проблема была бы обнаружена раньше
2. **Валидация данных** — добавить проверку наличия обязательных полей (`task_id`) при генерации

## 5. Было / Стало

**Было:**
- `task_id` терялся при передаче данных
- Блок "Задачи с низким баллом" пустой
- JavaScript фильтровал все задачи из-за отсутствия `task_id`

**Стало:**
- `task_id` корректно передаётся через всю цепочку
- 8,892 проблемных задач будут отображены
- Каждая задача имеет кликабельную ссылку на Pyrus: `https://pyrus.com/t#idXXXXXXX`

## Логика блока "Задачи с низким баллом"

**Критерий попадания:**
- Категории с maxScore = 14: оценка **≤ 13**
- Категории с maxScore = 18: оценка **≤ 16**

**Формат ссылки:**
```javascript
const pyrusUrl = 'https://pyrus.com/' + task.taskId;
// → https://pyrus.com/t#id123456789
```

