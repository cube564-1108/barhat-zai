## 📋 Как работать с дашбордом (правила)

### Правило 1: Перед любыми изменениями — бэкап

```bash
# Копируем MASTER
cp florist-quality-dashboard-MASTER.html florist-quality-dashboard-backup.html
```

### Правило 2: Одно изменение — одна проверка

1. Внёс изменение в скрипт
2. Сгенерировал HTML
3. Открыл в браузере — проверил
4. Только потом следующий шаг

### Правило 3: Не трогать работающее без необходимости

Если нужно **маленькое изменение** (текст, цвет, заголовок):
- Лучше отредактировать HTML напрямую
- Не перегенерировать через Python

Если нужно **большое изменение** (новые данные, новая логика):
- Использовать правильный скрипт генерации
- Проверить результат

## 🔧 Правильный процесс генерации

### Для новых данных из Pyrus (ЕДИНСТВЕННЫЙ способ):

```bash
# ТОЛЬКО этот скрипт для генерации из Pyrus API
python scripts/generate_dashboard_from_pyrus.py
```

**⚠️ ВАЖНО:** После генерации обязательно скопируй в MASTER:
```bash
cp florist-quality-dashboard.html florist-quality-dashboard-MASTER.html
```

### Для проверки результата:

1. Открой `florist-quality-dashboard.html` в браузере
2. Открой консоль (F12) — нет ли ошибок JavaScript
3. Проверь все блоки:
   - KPI карточки
   - Рейтинг салонов
   - Карточки салонов
   - **Задачи с низким баллом**

### Если что-то сломалось:

```bash
# Откат к MASTER
cp florist-quality-dashboard-MASTER.html florist-quality-dashboard.html
```
