# Workflow: Git + Amvera

> Стандартный процесс работы с кодом от локальной разработки до деплоя

## Общий процесс

```
[Локальная разработка] → [GitHub] → [Верификация] → [Деплой по команде]
```

## Фаза 1: Локальная разработка

1. Создаём/правим код локально
2. Частые коммиты (каждый осмысленный шаг)
3. Показываю результат владельцу → получаю фидбек
4. Переделываем если нужно

## Фаза 2: GitHub

1. Пуш в feature-ветку (НЕ в main)
2. Pull Request
3. Обсуждение, правки
4. Merge в main после одобрения владельца

## Фаза 3: Верификация

Перед деплоем:
- `/code-review` — поиск багов и чистки
- `/verify` — проверка что работает
- `/security-review` — если нужны критичные изменения
- Финальное "Ок" от владельца

## Фаза 4: Деплой на Amvera

1. Владелец даёт команду "Деплой"
2. Нажимаем кнопку "Deploy" в Amvera Dashboard
3. Ждём пока задеплоится
4. Проверяем что сайт живёт

## Важно

- **Никаких авто-деплоев** — только после команды владельца
- **Коммитим часто** — между этапами правок
- **Работаем в ветках** — main только для стабильного кода

## Стек

- **GitHub** — хранение кода
- **Amvera** — хостинг/деплой
- **Ручной деплой** — по команде владельца
