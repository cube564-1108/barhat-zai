# Agents — определения агентов

> Агент = специализированный AI-воркер для конкретной задачи.
> Отличие от скилла: агент — это код (Python/JS), скилл — это промпт.

## Типы агентов

| Тип | Роль | Когда нужен |
|---|---|---|
| Оркестратор | Координирует других агентов, собирает результаты | При задачах из 3+ фаз |
| Исследователь | Собирает данные, не принимает решений | Мониторинг, парсинг |
| Валидатор | Проверяет результат другого агента | После исполнителя |
| Синтезатор | Объединяет данные от нескольких агентов | Для дайджестов |
| Исполнитель | Выполняет конкретное действие | Отправка, запись, публикация |

## Промпт оркестратора

```
"Ты оркестратор реализации. Прочитай план [файл].
На каждую фазу пиши мне готовый промт — я буду реализовывать в соседнем окне.
Когда фаза закончена — жду проверки. Если ок → промт следующей фазы.
Степ-бай-степ до конца."
```

## Создать нового агента

```
Промпт:
"Создай агента [тип: исследователь/валидатор/исполнитель].
Задача: [что делает].
Источники: [откуда берёт данные].
Выход: [что возвращает].
Деплой: [Amvera worker / локальный скрипт]."
```

## Правило параллельности

**Параллельно запускать** только если агенты работают с разными файлами и данными.
**Последовательно** — если агент B зависит от результата агента A.

Перед параллельным запуском — спросить оркестратора: «Что из этих фаз можно параллельно?»
