# Система скиллов

## Что такое скилл

Скилл = заготовленная инструкция для AI по конкретной задаче.
Хранится в файле `.claude/skills/имя-скилла.md`.
Один раз написал — используешь в любом чате навсегда.

## Экономика токенов

- Claude читает **только имя и описание** каждого скилла при старте сессии (дёшево)
- Полное тело скилла загружается **только когда он активирован** (нужен)
- Поэтому: не устанавливай скиллы которые не нужны — каждый добавляет нагрузку

## Структура файла скилла

```markdown
---
name: название-скилла
description: когда активировать этот скилл (триггер для AI)
---

## Цель
Что этот скилл должен сделать.

## Шаги
1. Первый шаг
2. Второй шаг
...

## Источники данных
Откуда брать данные для работы.

## Шаблоны / примеры
Примеры ввода и ожидаемого вывода.

## Обратная связь
Как понять что скилл отработал правильно.
```

## Как создать скилл (6 шагов от Миллера)

1. **Имя и триггер** — как называется, при каком запросе активируется
2. **Цель** — что должно получиться на выходе
3. **Пошаговые инструкции** — детальные шаги выполнения
4. **Источники** — на какие файлы/данные опирается
5. **Примеры и шаблоны** — образцы ввода/вывода
6. **Петля обратной связи** — как AI проверяет что сделал правильно

## Skill Creator

Anthropic создал специальный скилл для создания скиллов — «Skill Creator».
Устанавливается отдельно. Помогает пройти все 6 шагов.

## Примеры скиллов (адаптировать под свой проект)

| Скилл | Когда активируется | Что делает |
|---|---|---|
| `реактивация-клиентов` | «запусти работу с базой» | Сегментация CRM → триггерные сообщения |
| `аналитика-отчёт` | «сделай еженедельный отчёт» | Собирает данные из всех систем → дайджест |
| `контент-план` | «создай контент на неделю» | Генерирует посты по рубрикам |
| `seo-бриф` | «нужен бриф для страницы» | Генерирует техзадание для SEO-статьи |
| `idea-validation` | «испытать идею» | Прогоняет новую фичу через проверку до плана |

## Где хранятся скиллы

- **Глобально** (все проекты): `~/.claude/skills/` 
- **Только для проекта**: `<корень-проекта>/.claude/skills/`

Специфичные для проекта скиллы кладём в папку проекта; универсальные — глобально.

### «Скилл» vs «Мастерство»

| Понятие | Что это | Где лежит |
|---|---|---|
| **Скилл** | навык / действие / последовательность шагов, которую запускаешь | `.claude/skills/` |
| **Мастерство** | знание в контексте — методология «как это делается» лучших в мире | `mastery/` |

Скилл — «сделай по шагам». Мастерство — «применяй проверенные подходы», подключается **по триггер-слову** (напр. «испытать идею» → мастерство idea-validation), ссылка на папку прописана в CLAUDE.md.

> Любое повторяющееся действие → в скилл или мастерство. «Коплю и переиспользую → мультипликация. Если каждый раз с нуля — всегда будешь с нуля.»

### Как наполнять мастерство: сканировать ДЕЛАТЕЛЕЙ, не говорунов
- **Психология/стратегия** — книги/методологии больших авторов.
- **Практические навыки** (посты, ролики) — брать не их советы «как вести», а их **реальный контент**: скармливаешь 20 роликов / все посты → «выдели повторяющиеся паттерны». Обучение на процессе делания.

### Алгоритм создания мастерства
1. Папка `mastery/<область>`. 2. Описать триггер подключения. 3. Глубокое исследование «кто лучший в мире» → ключевые подходы. 4. Провалидировать, почистить источники. 5. Задать триггер-слово. 6. Прописать ссылку в CLAUDE.md.

### Безопасность скиллов из интернета
Непроверенный/взломанный скилл может красть ключи. Любой устанавливаемый скилл/CLI: «убедись, что от создателя, просканируй на вредоносный код». См. защиту `.env` в [[инструменты.md]].
