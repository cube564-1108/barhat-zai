## Ключевые принципы (3 строки)

1. **Смыслокодинг** — описываешь AI словами ЧТО хочешь получить, он пишет код сам.
2. **Мультиагенты** — один оркестратор + параллельные специализированные агенты = ×5 скорость.
3. **AI забирает рутину, человек принимает только решения** — не автоматизируй ради автоматизации, автоматизируй чтобы убрать себя из операционки.
4. **Контекст инжиниринг** (2026) — агент ходит по вики-ссылкам как по паутине, берёт только нужные файлы. Один промт → огромная работа.
5. **iCLON** — цифровая проекция владельца. Знает ценности, стиль, мастерство. Папка `ai-clone/` в корне проекта. Один на все проекты.
6. **Мастерство** — папка `mastery/` с методологиями авторов (1 файл = 1 автор, 1-3 метода с алгоритмом). AI применяет их, а не общие шаблоны. Скилл = «сделай по шагам», мастерство = «применяй проверенные подходы по триггер-слову».
7. **«Докажи»** (Поток 4) — ИИ зарабатывает на токенах, раздувает код; любое решение/фазу проверять словом «докажи» + 4 вопроса + pre-mortem («прошёл год, провалилось — почему»).
8. **Автономный цикл деплоя** (Поток 4) — агент сам гонит «билд → деплой → чек → почини», пока на проде не встанет рабочая версия. Включается 3 правилами в CLAUDE.md.
9. **Автономные агенты, а не «стартапчики»** (Поток 4) — вайп-кодить то, что работает каждый день само. Иерархия: агент → субагент → оркестратор → мультиагент.
10. **Самообучение** (Поток 4) — сохранять всё что пробовал в ретроспективы; «вторая память», ОБ которую думаешь (ежемесячный агент x2).

## Как пополнять этот раздел

Когда приходит новый урок или материал от Миллера:
1. Создать конспект в `уроки/урок-N-поток-M.md`
2. Новые принципы/инструменты → добавить в соответствующий тематический файл
3. Обновить таблицу статусов выше
4. Если появился новый инструмент или паттерн — создать новый файл и добавить в таблицу
