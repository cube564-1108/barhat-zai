## Рабочая таблица: паттерн → как применить

| Паттерн | Доказательство (его цифры) | Как применить |
|---|---|---|
| **Робот-звонилка по лидам сразу после заявки** | конверсия ×2, маркетинг в 2 раза дешевле | автозвонок после заявки/брошенной корзины → перевод на оператора |
| **AI первая линия — человек финал** | 6000 AI-звонков/день по 30 коп/мин; оператор только с теми, кто сказал «да» | агент пишет/звонит всем, человек — только с откликнувшимися |
| **SEO-отдел из автономных агентов** | 3 глубоких статьи в день, цель 300 тыс. посетителей/день | агенты генерят SEO-статьи под выдачу ниши → бесплатный органический трафик |
| **Компания как код** | портал 10 000+ регистраций управляется через Claude Code, не через админку; 250 кронов | управление бизнесом через агентов/дашборд вместо ручной работы в интерфейсах |
| **Единый дашборд-оркестратор над агентами** | «офис» агентов управляет всеми бизнесами; цель — 1000 агентов | центр управления всеми агентами проекта |
| **Один источник → много форматов** | ссылка на видео → транскрипт + сценарий + посты | один набор данных → несколько видов контента |
| **Мониторинг + алерты** | парсер рекламных кабинетов алертит о сливе бюджета | метрики → алерт при отклонении, человек реагирует только на исключения |
| **Запускай MVP быстро, улучшай итерационно** | продукт за вечер по себестоимости токенов | первая версия → смотрим метрику → улучшаем |

## Метод валидации идеи перед запуском

Алгоритм проверки своей идеи (не для копирования чужого продукта):
1. Найти похожий проект, который «летит ракетой» (особенно зарубежный) → изучить вирусность и устройство.
2. Сделать своё **лучше** (своя архитектура, вау-эффекты), а не копию.
3. Просчитать бизнес-модель, маркетинг, математику (окупаемость, LTV, retention, налоги).
4. Обкатать на локальном рынке → масштабировать на международку.

*(Метод проверки идей лучше оформить как мастерство `idea-validation` — см. [[скиллы.md]].)*
