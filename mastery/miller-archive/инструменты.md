# Инструментальный стек Миллера

## Основной стек

| Инструмент | Роль | Стоимость |
|---|---|---|
| **VS Code** | IDE-оболочка, файловый менеджер проекта | Бесплатно |
| **Claude Code extension** | Основной AI-инструмент кодинга (официальный от Anthropic) | Бесплатно (расширение) |
| **Claude Opus 4.6** | Модель: 1M контекст, глубокое рассуждение | $200/мес (~15 000 ₽) |
| **Super Whisper / Visper Flow** | Голосовой ввод → текст, ×3 к скорости | Платные, небольшие |

## Настройки Claude Code (обязательно)

- **Bypass Permissions** — Claude действует без запроса разрешения на каждое действие. Без этого Claude спрашивает «можно ли?» перед каждым шагом — медленно.
- **Max Effort** — максимальное рассуждение перед ответом. Включать всегда.

## Почему Opus, а не Sonnet/Haiku

| Модель | Контекст | Когда использовать |
|---|---|---|
| **Opus 4.6** | 1M токенов | Основная работа — всё, что требует глубины |
| **Sonnet** | 200K токенов | Баланс цена/качество для несложных задач |
| **Haiku** | Маленький | Только маршрутизация, дёшево, не для сложного |

Миллер использует Opus для всего — не экономит на модели, экономит на количестве итераций.

## Голосовой ввод

- **Super Whisper** (Mac) — говоришь в микрофон → текст вставляется в любое поле
- **Visper Flow** — альтернатива
- Эффект: ×3 к скорости по сравнению с печатью
- Миллер ставит задачи голосом, не печатает

## Работа со скриншотами

- **Command+4** (Mac) → скриншот сохраняется в буфер обмена
- Вставляешь напрямую в чат Claude — он видит изображение
- Применение: показать ошибку, интерфейс, таблицу, дизайн

## Альтернативы которые Миллер НЕ использует

- Cursor — платный, уступает Claude Code по глубине
- Windsurf — аналогично
- Antigravity — аналогично
- GitHub Copilot — другая парадигма, не для смыслокодинга

## Хостинг и деплой

### Международный — Railway
- Всё в одной экосистеме: сайт + база данных + Redis + воркеры
- Подключается к GitHub → автоматически разворачивает при push
- ~$20/месяц за базовый стек

### Российский — Amvera
- Серверы в России → соответствует ФЗ-152 (персональные данные клиентов)
- Встроенный прокси для Claude и OpenAI → API работает без VPN с российских IP
- Есть PostgreSQL, Redis, Cron Jobs, воркеры
- Тот же принцип: GitHub push → автодеплой
- **Если проект хранит персональные данные россиян — всегда РФ-хостинг** (Amvera / Яндекс)

### Яндекс.Облако (Миллер использует для smyslokod.ru)
- Для крупных проектов с большой нагрузкой
- Дороже, больше контроля, ~10 000+ ₽/мес

## Базы данных

### PostgreSQL — «Excel в облаке для сайта»
- Хранит: пользователи, email, телефоны, заказы, оплаты, результаты проверок
- Нужна практически в любом проекте
- Claude создаёт и подключает сам по команде

### Redis — «Оперативная память для сайта»
- Управляет очередями запросов (1000 человек одновременно → ставит в очередь, не теряет данные)
- Rate limiting — защита от перегрузки и атак
- Кэширование для скорости
- Обязателен при массовой рассылке по базе клиентов

### Воркеры (Workers) — фоновые процессы
- Отдельные процессы для ботов, рассылок, AI-агентов
- Если воркер падает — основной сайт продолжает работать

## Workflow деплоя

```
1. Пишем/изменяем код в VS Code + Claude Code
2. Говорим Claude: "commit и deploy"
3. Claude коммитит (сохраняет снимок, описывает изменения)
4. Пушит в GitHub (отправляет в облако)
5. GitHub автоматически триггерит деплой на Amvera/Railway
6. Сайт/бот обновляется — готово
```

```
│   ├── life-metrics.md — LIVE-маркеры вместо хардкода цифр
│   └── ...
└── plans/              ← планы функций
```

### Life Metrics — живые данные из БД
- Файл `business/life-metrics.md`: как подключиться к API + маркер **LIVE** у каждой метрики
- API-эндпоинт read-only, обезличенные данные
- Агент сам тянет актуальные цифры вместо чтения устаревших markdown-файлов

### Звуковые хуки в settings.json
- **Stop hook** — звук при завершении задачи
- **Notification hook** — звук при запросе разрешений
- macOS: `afplay /System/Library/Sounds/Ping.aiff`

### Правила безопасности в settings.json (секция permissions.deny)
- `rm -rf` и любое массовое удаление без явного подтверждения
- `export` + `curl` — отправка переменных окружения наружу
- Чтение `.env` файлов и файлов с секретами
- `wget` с подставленными переменными

## Дополнительные инструменты

### Claude for Chrome — браузерная автоматизация
- Официальное расширение Claude в Google Chrome
- Режим «act without asking» — действует без подтверждения каждого шага
- Claude полностью управляет браузером: кликает, заполняет формы, прокручивает, сохраняет
- Применение: настройка сервисов (Amvera, GitHub, Яндекс.Метрика), мониторинг через браузер

### Open Router — маркетплейс AI-моделей
- Один ключ, одна оплата → доступ ко всем AI (Claude, GPT, Gemini и бесплатные модели)
- **Не блокирует российские IP** (Anthropic прямой API блокирует)
- API-ключ Open Router = заменитель ключа Anthropic в коде
- Есть бесплатные модели для недорогих задач

### Несколько ИИ в одном VS Code
В одно окружение встраивается несколько ИИ одновременно, все работают с одними файлами. Кончились лимиты у одной — переключаешься на другую. Расклад: **Opus** — архитектура и планы, **Qwen** — реализация (экономия токенов), **Codex/Antigravity** — поиск ошибок, **Claude** — исправление.

### Qwen Coder / Qwen 3.7 Max — китайская альтернатива Claude Code
- В 3–5 раз дешевле Opus (~$30/мес → лимит как у Claude за $100); отстаёт от Opus на 3–4%.
- Автономная работа до 35 часов с субагентами; контекст 1M.
- Оплата зарубежной картой, но **без VPN**.
- На security review сильнее Claude (находит уязвимости, что пропускают GPT-5/Claude); **Kimi 2.6** — аналогично. → второй проход security-review.
- **DeepSeek** подключается, но не понимает картинки (Claude и Qwen — понимают).

### 5 режимов permissions + Bypass по умолчанию
Спросить перед edit / почти авто / план-мод / best mode / **Bypass** (полная автономия). Сделать дефолтным: extension → Claude Code → Settings → галочка «разрешить bypass mode» → новое окно.

### Защита `.env` хуком
Мало `.gitignore` — нужен **хук, физически запрещающий ИИ читать `.env`**: непроверенный/взломанный скилл может украсть ключи. Любой устанавливаемый скилл/CLI проверять: «от создателя ли, нет ли вредоносного кода».

### Хостинг — уточнения
- **Railway** — мощнейший, но **free-тарифа НЕТ**. **Vercel** — есть free.
- **Amvera** — push в git + конфиг, проксирование моделей за рубли без зарубежной карты. **TimeWeb** — аренда серверов. **Яндекс Cloud** — крупные нагрузки.
- Аргумент за РФ-хостинг помимо 152-ФЗ: РКН может заблокировать целого зарубежного хостера (если на тех же IP развернули VPN) → сайт перестаёт открываться у россиян.
- `.env` при деплое клод сам доставляет на сервер через CLI.

> **Про модель:** актуальны **Opus 4.7 / 4.8** — версии сменяются быстро, ориентироваться на самую свежую Opus.
