# Управление контекстом

## Что такое контекстное окно

Claude держит в «голове» ограниченный объём текста — это контекстное окно.
- Opus 4.6: 1 миллион токенов (~750 000 слов)
- Sonnet: 200 000 токенов

Когда окно заполняется — AI начинает «забывать» начало разговора.
Поэтому нужно управлять тем, что занимает пространство.

## Что занимает токены (от большего к меньшему)

1. История диалога (накапливается по мере общения)
2. MCP-серверы (каждый добавляет 50–100K токенов просто фактом подключения)
3. Файлы которые AI читает (чем больше файл, тем дороже)
4. CLAUDE.md (читается каждую сессию — держи до 200 строк)
5. Скиллы — только описание при старте, тело только при активации

## Правила экономии

- **Одна задача = один чат.** Не смешивай разные задачи.
- **CLAUDE.md ≤ 200 строк.** Больше — расточительно.
- **Wiki/index.md — сжатая карта, не копия.** Полные данные в подфайлах.
- **MCP — только нужные.** Не подключай все подряд.
- **Скиллы — только актуальные.** Убирай неиспользуемые.

## Команды управления контекстом

| Команда | Действие |
|---|---|
| `/compact` | Сжимает историю диалога, освобождает контекст |
| `/clear` | Очищает историю полностью (осторожно — теряешь контекст сессии) |

## MCP-серверы: USB для сервисов

MCP (Model Context Protocol) = протокол подключения внешних сервисов к Claude.

Что можно подключить:
- Google Sheets — Claude читает/пишет таблицы напрямую
- Google Calendar — работа с расписанием
- Управление браузером — Claude сам кликает и заполняет формы
- CRM-системы — прямой доступ к клиентской базе
- Stripe — платёжные данные

**Важно: каждый MCP дорог по токенам.** Подключать только то, что реально используешь.

Приоритетные MCP под типовой проект:
1. **CRM** — если есть API-доступ (прямая работа с базой клиентов)
2. **Google Sheets** — аналитический дашборд
3. **Браузер** — скрапинг, мониторинг сайта

## Стратегия загрузки файлов

**Всегда загружается:**
- CLAUDE.md (навигация)
- .claude/skills/ описания (только имя + описание)

**Загружается по задаче:**
- .business/wiki/index.md (когда нужен бизнес-контекст)
- Конкретные файлы из .business/ (когда нужна детализация)
- plan-файл (когда работаем над конкретной функцией)

**Загружается только при активации:**
- Полное тело скилла

## Практическое правило Миллера

> «Открыл новую задачу — новый чат. Сделал задачу — написал итог в план-файл. Следующий чат начинается с чистого контекста но с актуальными файлами.»

Это как оперативная память компьютера: чем меньше открытых вкладок, тем быстрее работает.

### «Второй мозг» = «вторая память», ОБ которую думаешь
База заметок, перелинкованных викилинками — не просто архив для чтения контекста, а инструмент решений:
> «Что ключевое сделать в этом месяце, чтобы x2 от прошлого?» — ИИ собирает накопленный контекст и предлагает решение, до которого сам не додумался бы.

### Почему контекст важнее «ума» модели
Gemini кажется тупой не из-за модели, а из-за слабого контекста. «Дело не в глупой модели, а в плохом контексте.» Цель контекст-инжиниринга — чтобы нейросеть предчувствовала, что мы хотим.

### Жёлтый индикатор контекста = неэффективная работа
Загорелся жёлтым — «пытаетесь в одном чате напихать всё». Закончил задачу → закрыл чат → открыл новый. Рабочий режим: несколько проектов × несколько чатов, звуковой хук сообщает о завершении.
