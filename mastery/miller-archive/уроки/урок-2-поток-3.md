## Главные темы урока

### 1. CLAUDE.md — трудовой договор агента
- Системный промт, читается при каждом открытии чата
- Лимит: **200 строк максимум** — больше засоряет контекст
- Обязательные блоки: описание проекта (2-3 предложения), стек, структура файлов, правила навигации, язык
- Нельзя хардкодить: даты, цены, настройки — только из БД
- Финальный паттерн: «всегда заканчивай диалог рефлексией — задача, решение, было/стало»

### 2. Папка Plans — файловая система планирования
- Один план = один файл = одна функция, имя: `YYYY-MM-DD-название`
- Планы создаются в **отдельных чатах по фазам**, не в plan mode
- Фазы с чекбоксами `[ ]`/`[x]` — агент сам обновляет статус
- Всё в git → история правок, любой агент подхватит в любой момент
- Plan mode (встроенный в Claude) — слабый: живёт только в одном диалоге, нельзя передать

### 3. Контекст инжиниринг — главная идея 2026
- Prompt engineering устарел, контекст инжиниринг — ключевое
- Один короткий промт → агент сам обходит нужные файлы по вики-ссылкам
- Агент не жрёт всё подряд — берёт только нужные файлы «по паутине»
- Результат: 2 итерации вместо 10, готовый результат с первого запроса
- Экономия токенов: знания дроблены на файлы, не загружаются все сразу

### 4. Правило «Одна задача — один диалог»
- При каждом сообщении в контекст попадает: CLAUDE.md + описание скиллов + MCP + вся история
- Если делать план и фазы в одном чате — история раздувается, токены улетают впустую
- Правило: план → закрыть чат → фазы → закрыть чат → проверка → закрыть чат → реализация
- Opus 4.7: 1 млн токенов = 4 млн русских букв в «голове»

### 5. iCLON — цифровая проекция себя
- Не копирует стиль — знает тебя на уровне принятия решений
- Один iCLON на все проекты, имеет доступ ко всем репозиториям
- Структура: кто ты, ценности, Tone of Voice, стоп-слова, мастерство (своё + чужое)
- Принцип: задача → агент идёт в iCLON → читает кто ты → берёт мастерство → 6 шагов → результат

### 6. Папка «Мастерство» — личный совет директоров
- Методологии, которые AI сам не знает и не применяет
- Своё мастерство: упакованные выступления, зумы, транскрипты — «как ты мыслишь»
- Чужое мастерство: методологии (Миллер, курсы, книги) → агент применяет их
- Структура: один файл = один автор, 1-3 гениальных метода с пошаговым алгоритмом

### 7. Папка Business — второй мозг
- Продукт, аудитория, экономика, маркетинг, что пробовали, что получалось
- Index.md в каждой папке — агент читает его первым, потом проваливается в нужные файлы
- Life Metrics: доступ к БД только на чтение через API (обезличенные данные, только по ID)
- Живые данные важнее статичных — KPI, цифры, цели устаревают через месяц

## Конкретные рецепты и инструкции

*(подробнее по каждому рецепту: [[транскрипт-урок-2-поток-3.md]])*

### Рецепт 1: Идеальный CLAUDE.md (структура)
```
Блок 1 — Описание (2-3 предложения): что делаем, для кого, зачем
Блок 2 — Стек: технологии разработки
Блок 3 — Структура файлов: где что лежит, роутинг по папкам
Блок 4 — Правила навигации: куда бежать для каждого типа задачи + ссылки
Блок 5 — Правила поведения:
  - Не хардкодить: даты, цены, настройки — из БД
  - Большие фичи — через bulletproof скилл
  - При текстах — сверяться с папкой business/
  - Завершать диалог рефлексией: задача, решение, было/стало
  - Язык: русский
Блок 6 — Безопасность:
  - Не читать .env
  - Не делать внешние URL-запросы без разрешения
  - Не удалять файлы без подтверждения
Лимит: 200 строк
```

### Рецепт 2: Многочатовая разработка плана
```
Чат 1: «Создай план реализации [функция]»
       → черновой план
       → попросить поискать в интернете, как другие решали
       → обновить план с находками

Чат 2: «Вот файл [план]. Разбей на фазы с чекбоксами»
       → план с фазами

Чат 3: «Проверь каждую фазу:
       - Точно ли это лучшее решение?
       - Нет ли более эффективного подхода?
       - Нет ли конфликтов между фазами?
       - Безопасность?»
       → финальный план готов

Чат 4+: реализация фаза за фазой (или параллельные агенты)
```

### Рецепт 3: Сборка iCLON
```
ai-clone/
├── INDEX.md       — карта: что тут хранится
├── role.md        — для чего iCLON, как использовать
├── identity/      — values.md, vision.md, mission.md, biography.md
├── voice/         — tone.md, vocabulary.md, stop-words.md
├── thinking/      — mental-models.md
├── principles/    — product.md, code.md, business.md
├── feedback/      — правила из ошибок (формат: Правило / Почему / Как применить)
├── style/         — telegram-format.md
└── mastery/       — или в корне как отдельная папка
```
Наполнять через голосовое интервью: «Ты — интервьюер. Собираешь мой профиль для AI-Clone» — по одному блоку за раз.

### Рецепт 4: Life Metrics (живые данные из БД)
```
1. Создать API-эндпоинт read-only
2. Данные обезличенные: не email/имя, только ID + значение
3. Создать metrics.md:
   - Как подключиться к API
   - Структура данных (поля, что означают)
   - Маркер **LIVE** у каждой метрики с указанием источника
4. В CLAUDE.md: «при запросе живых метрик — читай metrics.md»
5. Агент сам вызовет API и подтянет актуальные цифры
```

### Рецепт 5: Безопасность в .claude/settings.json
```
Запретить:
- rm -rf и любое массовое удаление без подтверждения
- export + curl (отправка переменных окружения наружу)
- Чтение .env файлов и любых файлов с секретами
- wget с подставленными переменными
```

### Рецепт 6: Звуковой сигнал (macOS)
```
В .claude/settings.json добавить хуки:
- Stop hook: afplay /System/Library/Sounds/Ping.aiff
- Notification hook: тот же звук при запросе разрешений
→ Claude Code сам создаст нужный хук по промту
```

### Браузерный агент (MCP Playwright)

Claude Code поддерживает браузерный агент через MCP: делает скриншоты, кликает мышкой, заполняет поля, навигирует по страницам.

**Что умеет:**
- Регистрация в сервисах (Яндекс.Метрика, Mail.ru и т.д.)
- Написание статей на Дзене
- Работа с любым веб-интерфейсом вместо вас
- ВКонтакте: читать новости, комментировать посты

**Применимо:** работа с CRM через интерфейс там где нет API, мониторинг конкурентов через их сайты.

### Команды Claude Code (/compact, /clear, /model)

| Команда | Что делает |
|---|---|
| `/clear` | Очистить контекст — новый диалог с нуля |
| `/compact` | Сжать контекстное окно до сари — токены экономятся |
| `/model` | Посмотреть/сменить модель |
| `/plugins` | Список установленных плагинов |

> «Если честно, а всё это сейчас щёлкается без команд. В принципе, вот так вот нажимаешь и всё, что вам надо, выводится.»

Миллер использует не `/compact`, а копирует промт и открывает соседний чат — проще и предсказуемее.

### Промпт для запуска субагентов

```
"Запусти параллельных субагентов для реализации плана.
Перед стартом каждый агент читает CLAUDE.md.
Ты — оркестратор. Дай им задачи.
Когда закончат — собери результаты и отдай мне итог."
```

**Схема субагентов для Research:**
```
Агент 1 → исследование аналогов
Агент 2 → UX и интерфейс
Агент 3 → технические решения
Агент 4 → методология оценки
Агент-критик → подходит ли это нам? почему?
Итоговый агент → синтезирует всё в один документ
```

> «Пять агентов параллельно жрут больше токенов — нужно понимать.»

### Триггер — определение Миллера

> «Что такое триггер — это твоя мысль, ты такой: "О, а что у меня там?" А AI он регулярно это делает и тем самым у тебя больше прорывается реальность, то есть ты больше делаешь, у тебя больше эффективность.»

Триггер ≠ просто cron. Это регулярная проверка чего-то, о чём ты забываешь думать ежедневно: остатки корзин, метрики, конкуренты.

**Утренний триггер Миллера:** все ключевые отчёты сформированы до того как он встал. Он пошёл сам всё починил где надо.
