## Главные темы урока

### 1. Полная архитектура смыслокодинга

Урок показывает **полный путь** — от идеи до работающего продукта в интернете. Сжатая версия за 2 часа эфира, полная — месяц в тренинге.

**Цикл большого проекта:**
```
Папка идеи (Research + конкуренты)
  → Папка архитектуры (стэк, БД, структура папок/файлов/данных)
    → Написание кода
      → Папка business (появляется позже идеи)
        → Папки идеи и архитектуры удаляются (устарели)
          → Триггеры и автоматизации
            → «Я = компания»
```

> Идея всегда приходит раньше, чем формируется бизнес. Поэтому бизнес-папка появляется позже.

**Малый vs большой проект:**
- Маленький → сразу по 6 шагам, без отдельной папки архитектуры
- Большой (SaaS, CRM, стартап) → обязательна папка `architecture/` до кода

### 2. Шестишаговая система

| # | Шаг | Суть |
|---|-----|------|
| 1 | **Research** | Deep Research: аналоги, мировые решения, доказанные практики |
| 2 | **Plan** | Составить план с фазами и чекбоксами, валидировать |
| 3 | **Реализация** | Код по фазам через оркестратор, каждая фаза → коммит |
| 4 | **Review** | «10 причин обосраться» — почему не залетит |
| 5 | **Commit** | GitHub, осмысленные сообщения коммитов |
| 6 | **Deploy** | Отправить в интернет (Vercel / Amvera) |

### 3. Оркестратор реализации ★ КЛЮЧЕВОЙ ПАТТЕРН

**Проблема:** план и реализация в одном чате → история раздувается → токены улетают → агент «жрёт» весь предыдущий контекст.

**Решение: два чата.**

```
Чат-оркестратор:          Чат-исполнитель:
  читает план               получает промт от оркестратора
  пишет промт фазы 1  →     реализует фазу
  ждёт результат      ←     отчитывается
  проверяет
  пишет промт фазы 2  →  ...
```

**Промпт оркестратору:**
> «Ты оркестратор реализации. Прочитай план. На каждую фазу пиши мне готовый промт — я буду реализовывать в соседнем окне. Когда закончена — прошу проверить. Если ок → промт следующей фазы. Степ-бай-степ до конца.»

*(подробнее: [[транскрипт-урок-3-поток-3.md]])*

### 4. Параллельные агенты — только при независимых фазах

Когда фазы **независимы**, можно запустить несколько чатов параллельно:
```
Агент 1 → реализует фазу 2
Агент 2 → реализует фазу 3  
Агент 3 (критик) → смотрит на результаты 1 и 2, пишет замечания
```

**Правило:** сначала спросить оркестратора — «что из этих фаз можно параллельно?». Не гадать самому.

### 5. Антипаттерн «каша»

Если сказать «сделай всё сам»:
- Агент не вмещает весь план в контекстное окно
- Запускает параллельных агентов бесконтрольно
- Они переписывают код друг друга
- Проект разваливается — починить невозможно

> «Чем больше проект — тем больше контроль требуется.»

### 6. Принцип «Я = компания»

> «Между вами нет прокладки в виде людей, зумов, программистов, которые по-другому понимают задачу. Теперь вы один можете реализовать всё, что в вашей голове.»

> «Раньше сотни тысяч рублей, месяцами встречи, в итоге не то, что хотели. Теперь за вечер по себестоимости 2000 рублей на токены.»

## Новые инструменты и паттерны

*(подробнее по каждому: [[транскрипт-урок-3-поток-3.md]])*

### Deep Research перед каждым проектом

Три инструмента параллельно:
- **Perplexity** Deep Research
- **Google** Deep Research
- **ChatGPT** Deep Research

→ Собрать результаты → проанализировать → выжать в `IDEA.md`

Промпт Research:
> «Это моя идея. Проведи глубокое исследование: кто в мире уже решал эту проблему, как реализовали, какие ключевые методологические смыслы. Найди только эффективные решения с доказанной эффективностью. Запусти параллельных агентов. Финальный агент синтезирует в один файл IDEA.md.»

### IDEA.md — артефакт Research

После Research создаётся `IDEA.md`:
- Выжимка лучших мировых решений по задаче
- Только доказанно работающие подходы
- Очищен от галлюцинаций (три итерации валидации)

**3 итерации валидации IDEA.md:**
1. «Учитывая цели — что здесь лишнее?»
2. «Будь критиком каждой идеи. Докажи почему хорошая — хорошая, плохая — плохая.»
3. «Напиши каждую идею человеческим языком: 1 строка суть + 1 строка зачем нам это»

### Коллективный разум — паттерн форков

Пример с Whisper Flow в тренинге:
1. Игорь сделал скилл для установки Whisper Flow
2. Другой участник допилил, решил проблемы
3. Третий форкнул, адаптировал для Windows
4. Четвёртый сделал платный продукт (Whisper за 800 рублей)

**Паттерн:** один создал → сообщество форкнуло и улучшило → кто-то монетизировал.

### Deploy GitHub → Vercel

Только для проектов **без хранения персональных данных РФ**.

Промпт:
> «Создай приватный репозиторий на GitHub, назови его [название]. Добавь все файлы кроме .env. Сделай первый commit.»

После → Vercel импортирует из GitHub → автодеплой при каждом push.

## Ключевые цитаты Миллера

> «Твой проект — это идея, глубоко проработанная с ресёрчем. Это далее архитектура этой идеи — чертёж дома. Потом код.»

> «Его задача — отдавать вам больше букв, чтобы вы за эти буквы заплатили. Поэтому он очень часто генерирует кучу бесполезного текста, раздувает контекстное окно.»

> «Вы получаете под капотом условно 10–20 разработчиков, которые ещё и двигаются там x10 по скорости.»

> «Чем больше проект — тем больше контроль требуется.»

> «Теперь вы все, что задумали, можете запросто превратить в код. И это будет работать. Это просто потрясающе.»

1. **GitHub + деплой** — приватный репозиторий, .gitignore со всеми секретами, автодеплой из main. Без этого проекта нет в интернете.

### 🟡 Ядро методологии (делать следующими)

3. **life-metrics агент** — тянет метрики (выручка, заказы, средний чек, новые клиенты) из API → пишет в `business/metrics/YYYY-MM-DD.md` → шлёт сводку в Telegram. Задача на 1 сессию.

6. **Структура `triggers/`** — собрать всех агентов в одну папку с README-реестром.

7. **Стресс-тест по 10 пунктам** — честный аудит проекта через линзу Миллера.
