## Главные темы урока

### 1. Путь localhost → ссылка в браузере

Локальный `localhost:8000` нельзя открыть извне: у домашних компьютеров нет статического IP (операторы закрывают внешний доступ, IP меняется). Поэтому нужен внешний сервер. Полный путь на живом примере (вчерашний проект): ноут → GitHub → деплой на сервер → публичная ссылка.

### 2. GitHub как «Яндекс.Диск для кода»

Бытовое объяснение для непрограммиста:
- **Репозиторий** = папка проекта в облаке.
- **Commit** = сохранение изменений (как Ctrl+S), упаковка куска кода с описанием «кто, когда, что и зачем».
- **Push** = отправка сохранений в облако.
- **Приватный** (ты + команда) / **публичный** (любой по ссылке скачает и поднимет у себя).
- Зачем: командная работа, обмен решениями, и главное — **история изменений для отката**.

### 3. .gitignore: «GitHub помнит всё» ★

`.gitignore` перечисляет файлы/папки, которые НЕ должны утечь (ключи из `.env`, папка business, локальные конфиги).
> «GitHub, как и интернет, помнит всё.»

Если чувствительные данные раз попали в репо — удаление не помогает, они остаются в истории. Защиту закладывать ЗАРАНЕЕ, до первого пуша. Если уже утекло — команда «удалить из репа + переписать историю».
> «Чаще всего, когда поимеют ваши ключи — имеют ваш проект. Не потому что он дырявый, а потому что ключи в паблик утекли.»

### 4. Commit / Reset / откат — версионирование как страховка ★ НОВОЕ

Болевая история Миллера: вайп-кодил 2 дня без коммитов, запустил параллельных агентов — они переписали код друг друга → «билиберда» → 2 дня коту под хвост. Вывод: коммитить часто.
На живом примере: добавил «летающие мозги» в проект → нашёл этот коммит по истории → reset (откат). В VS Code раздел Source Control показывает коммиты, что задеплоено и что нет (разными цветами).

### 5. Выбор сервера: РФ vs международный

| Тип | Сервисы | Заметки |
|---|---|---|
| **Международные** | Railway (мощнейший, но платный — нет free), Vercel (есть free), аналоги | для проектов без перс. данных РФ |
| **РФ** | Amvera (push в git + 1 конфиг), Яндекс Cloud (крупные нагрузки, там Смыслокод), TimeWeb (аренда серверов) | 152-ФЗ |

- **152-ФЗ:** персональные данные россиян нельзя на зарубежных серверах. Сам VPS может быть зарубежным, но **БД — российская**. Лучше БД и серверы в одной экосистеме.
- **Аргумент про блокировки:** в гонке блокировок могут заблокировать целого зарубежного хостера (если кто-то развернул на тех же IP VPN) → твой сайт перестаёт открываться у россиян. Для РФ-проектов — хостить на РФ (Яндекс — высокий «траст»).
- **Amvera:** проксирование моделей (GPT-5 и др.) с оплатой в рублях без зарубежной карты.
- **`.env` при деплое:** клод сам доставляет переменные на сервер через CLI — вручную не надо.

*(детали → [[инструменты.md]])*

### 6. Автономный цикл «билд упал → почини → задеплой» ★★ главная тема урока

Раньше: билд падал → «найди ошибку» → чинит → ты вручную снова деплоишь — итеративно руками. **Сейчас, с думающими моделями, агент делает это сам в цикле**: собирает → билдит → деплоит → чекает → исправляет → повторяет, пока на сайте не встанет рабочая версия.
> «Если упало — он сам чинит. 90% случаев. В код лезть не надо.»

Реализуется правилом в `CLAUDE.md` (см. промпты). Год назад Миллер не смог побороть одну ошибку даже с бэкграундом программиста — сейчас «очень-очень легко».
*(детали → [[процесс.md]])*

### 7. «Мастерство» vs «Скилл» — разграничение ★

- **Скилл** = навык / действие / последовательность шагов, которую запускаешь (напр. скилл «мастерская гайд» пишет большую SEO-статью).
- **Мастерство** = знание в контексте, набор методологических знаний «как это делается», лежит в папке `mastery`. Подключается по триггер-слову.
- Любое повторяющееся действие → в скилл или мастерство. «Коплю и переиспользую всё → мультипликация, экспонента. Если каждый раз с нуля — всегда будешь с нуля.»

*(детали → [[скиллы.md]])*

### 8. Как наполнять «мастерство»: сканировать ДЕЛАТЕЛЕЙ, не говорунов ★ НОВОЕ

Различать тех, кто *рассказывает* как надо (часто инфоцыгане без результата), и тех, кто *делает* с подтверждёнными цифрами. Для практических навыков (посты в TG, YouTube) брать не их советы «как вести», а их реальный контент: скармливаешь 20 роликов / все посты → «выдели ключевые повторяющиеся паттерны». Обучение на процессе делания, а не на разговорах.

### 9. iCLON — один на все проекты

> «Я — одна сущность, зачем меня клонировать на каждый проект.»

Один iCLON в текущем/центральном проекте. Наполняется личными дневниками, записями, рефлексиями. Содержит цели на жизнь, проекты, ключевую информацию о владельце.

### 10. AI-Cofounder / центр управления компанией ★ НОВОЕ

Проект **AI Founder** (aifounder.ru) — визуальный «офис» с человечками-агентами. Под капотом не игра и не n8n, а **код-обёртка вокруг Claude Code (CLI), которая запускает субагентов как рутины**. Через Claude Desktop лимит 15 рутин — поэтому своя оболочка. Это «центр управления всеми бизнесами» / генеральный директор. Каждый агент = методология + расписание. Цель года — 1000 автономных «сотрудников».

### 11. Дизайн через «методологию дизайна» в CLAUDE.md

Создать файл «методология дизайна» (цвета, стиль, правила) + ссылку на него в `CLAUDE.md`, чтобы каждая новая страница вайп-кодилась в фирменном стиле. Свой стиль (оранжевое свечение) Миллер «спёр» из интернета и оптимизировал. Про Claude Design: лишняя трата времени для вайп-кодера. «Продают смыслы, а не красивая обёртка.»

### 12. Контекст и «почему Gemini тупая»

ChatGPT хорошо закладывает контекст (спроси «5 фактов обо мне» — угадает), Claude недавно начал на телефоне, Gemini почти нет — оттого кажется тупой. «Дело не в глупой модели, а в плохом контексте.» Аналогия: контекст-инжиниринг — чтобы нейросеть предчувствовала, что мы хотим.

### 13. Наложение методологии на существующие проекты

Не переписывать с нуля. Дать клоду текст методологии и «давай step-by-step»: создай папку `business`, просканируй код, собери ТОЛЬКО ключевые смыслы (не всё подряд — иначе раздуется). iCLON — личными дневниками.

## Новые инструменты, сервисы, модели

- **Railway** — международный хостинг, «самый мощный», полный набор БД/Redis/серверов; клод через CLI сам создаёт базы и серверы. **Free-тарифа НЕТ.**
- **Vercel** — международный, есть free; на живом примере клод сам сказал, что плохо подходит для тяжёлого проекта, предложил Railway.
- **Amvera** — РФ, деплой push в git + конфиг; есть официальный Amvera CLI (Миллер заставил клода проверить подлинность и просканировать на вредоносный код); проксирование моделей за рубли.
- **TimeWeb** — РФ, аренда выделенных серверов.
- **Яндекс Cloud** — для крупных РФ-проектов (там Смыслокод).
- **Qwen / Qwen Coder (Alibaba)** ★ — «шайтан-машина», почти идентичен Claude Code. ~$30/мес → лимит как у Claude за $100. Нашёл **4 уязвимости** на security review, которые не нашли ни GPT-5, ни Claude. Минус: 10 параллельных агентов зависают (временное).
- **Kimi 2.6** — китайская модель: 10 агентов нашли **46 уязвимостей** (сильнее Claude в этом тесте).
- **Claude Opus 4.8** — вышел в день эфира.
- **Nano Banana** — генерация картинок; текст рисуется отдельно в HTML и соединяется с картинкой (чтобы не было ошибок в русском).
- **MCP** — «USB-хаб»; **API — это ранняя версия MCP** (упрощённо).
- **Bitrix24 API** — подключить к ИИ, тянуть данные для анализа конверсии звонков.
- `/login` в терминале VS Code — при ошибке авторизации.

## Конкретные промпты (самое ценное)

**Три критические строчки в CLAUDE.md — ядро «доведения до результата»:**
1. «Никогда не отправляй в деплой моё приложение, пока я тебе не разрешу.»
2. «Всегда делай коммит — коммить только свои изменения, не трогай чужие.»
3. «Когда отправляешь в деплой — пока не убедишься, что деплой прошёл и на сайте актуальная версия: отслеживай и находи ошибку, переделывай, обновляй, лечи, пока до конца не задеплоишь.»

**Публикация в GitHub:**
> «Создай GitHub-репозиторий, назови как проект, закоммить все изменения и отправь, сделай публичным. Очень важно — ключи из `.env` ни в коем случае не отправляй в публичный репозиторий, поставь себе блок. Добавь в gitignore папку business.»

**Откат / поиск бага по истории коммитов:**
> «Там какие-то мозги появились, не знаю когда это вышло — полазь по коммитам, найди момент когда это произошло.» → «Откати мне эти изменения, не нравятся.»

**Деплой на Amvera:**
> «Amvera — российский хостер, найди возможность подключаться, чтобы ты полностью делал все настройки сам без меня: загрузил проект, отправил в жизнь — всё что угодно после того, как я авторизуюсь. Скорее всего есть готовое решение, найди его.»

**Проверка безопасности устанавливаемого CLI/скилла:**
> «Убедись, что то что ты установил — правда от создателя, чтобы не наткнуться на скамера, и обязательно проверь на предмет вредоносного кода.»

**Создание мастерства (валидация идей):**
> «В папке iClone/мастерство создадим первое мастерство, которое используем по умолчанию когда сталкиваемся с задачей: когда я новую фичу/идею хочу написать — ты всегда подключаешь это мастерство, которое идею сразу испытывает на адекватность. Проведи глубокое исследование: кто лучше всего в мире выделил методологические подходы проверки гипотезы перед реализацией. Найди ключевые подходы.»

**Привязка триггер-слова:**
> «Сделай триггер-слова, по которым мастерство запускается. Пусть будет „испытать идею“.»

**Создание методологии дизайна:**
> «Создай файл методология дизайна и в CLAUDE.md пропиши ссылку, что когда создаём страницу/фичу с дизайном — всегда обращаемся к этому файлу: какие цвета, стиль, правила.»

**CRM с нуля вместо готовой:**
> «Найди опенсорс-решение для организации продаж (CRM, похожее на RetailCRM). Возьми, у меня вот такой бизнес — убери всё лишнее, оставь важное.»

**Масштабирование при выборе тарифа:**
> «Планирую 5000 пользователей — заранее что подготовить, но сейчас денег не тратить, быть готовым к росту.»

## Пошаговые техники

### Путь «localhost → ссылка»
1. Делаешь проект локально. 2. Создаёшь GitHub-репо, коммитишь, пушишь. 3. ДО пуша настраиваешь `.gitignore` (ключи, business). 4. Говоришь клоду подключить репо к Vercel/Amvera/Railway, дать ссылку. 5. Авторизуешься на сервере. 6. Клод билдит, деплоит, отдаёт ссылку. 7. Дальше цикл: «commit» → «deploy» → обновляется прод.

### Автономный цикл доведения деплоя
1. Отправляешь в деплой. 2. Билд упал → клод сам читает логи → находит ошибку → чинит → снова билдит/деплоит. 3. Чекает, что встала актуальная версия. 4. Повторяет, пока не задеплоится (правило в CLAUDE.md «пока не задеплоишь в прод — не успокаивайся»).

### Поиск и откат бага через коммиты
1. Заметил нежелательное изменение. 2. «Найди коммит, где появилось». 3. Смотришь diff (было/стало). 4. «Откати» → reset. 5. Коммитишь откат.

### Создание мастерства
1. Папка `mastery` (внутри iClone). 2. Описываешь триггер. 3. Глубокое исследование «кто лучший в мире» → ключевые подходы. 4. Валидируешь, чистишь источники. 5. Задаёшь триггер-слово. 6. Прописываешь ссылку в CLAUDE.md.

### Валидация бизнес-идеи перед кодом (его процесс на примере Pirat.x)
1. Находишь похожий проект, который «летит как ракета» (особенно зарубежный). 2. Изучаешь вирусность, устройство, промпты. 3. Делаешь своё лучше — своя архитектура, вау-эффекты, сканеры. 4. Простраиваешь бизнес-модель, маркетинг, математику. 5. Заливаешь трафик, смотришь окупаемость, LTV/retention, рентабельность с налогами. 6. Работает на РФ → масштабируешь на международку (×1000 по ёмкости).

## Ключевые цитаты

1. «Я предприниматель в первую очередь, программист во вторую… у меня всегда на первом месте методологический подход. Код — в последнюю очередь, потому что код — всего лишь проекция нашего решения.»
2. «GitHub, как и интернет, помнит всё. Лучше заранее заложить отсутствие этого процесса.»
3. «Чаще всего, когда поимеют ваши ключи — имеют ваш проект.»
4. «Когда он говорит, что это надо делать день, два, месяц — не верьте, он оценивает, если бы это делал человек. Мы сделаем за пять минуточек.»
5. «Если упало — он сам чинит. 90% случаев. Вам не надо лезть в код.»
6. «Дружище, пока не задеплоишь в прод — не успокаивайся, ищи ошибки, исправляй и отправляй обратно в деплой.»
7. «Когда мы просыпаемся — всё, что мы навайп-кодили, и мастерство, работает на нас. Если ничего не копить — будешь всегда с нуля.»
8. «Весь интернет захламлён говном, которое навайп-кодено и не работает. Задача — навайп-кодить что-то реально полезное.»
9. «Не надо бегать всё подряд вайп-кодить — будет много начато и ничего не закончено. А всего лишь одну ключевую функцию, которая меняет правила игры жёстко.»
10. «Перестаёшь быть оператором — становишься архитектором.»
11. «Я не продаю результат, я даю знание и квалификацию — методологию, способ мышления и действия, чтобы повысить вероятность наилучшего результата.»
12. «Люди покупают, когда удивляются и восхищаются.»
13. «Если у тебя на цифрах результаты — огонь. Победитель не судит. Не надо переубеждать собственника — сделай цифры, он сам поймёт.»
14. «Как только возьмёте в команду вайп-кодера — никогда не захотите других людей. Через полгода каждый хороший бизнес будет иметь вайп-кодеров в штате.»
15. «В новом деле никогда нельзя заканчивать старое, особенно если оно приносит результат — продолжай работать со старым и добавляй новое.»

## Домашнее задание

Формального ДЗ нет (последний день практикума, упор на продажу тренинга). Практическое задание для повторения:

2. Задеплоить на внешний сервер (Vercel/Railway — международные; Amvera/Яндекс — РФ), получить рабочую ссылку.

5. Опционально: файл «методология дизайна» с фирменным стилем.
6. Поставить Qwen Coder при проблемах с Claude Code.

## 🎯 Чеклист применения по обоим урокам Потока 4

### Базовая дисциплина (любой проект)
1. **Автоцикл деплоя в CLAUDE.md** — правило: «при деплое отслеживай билд, читай логи, находи и чини ошибку, повторяй пока не встанет актуальная версия» → агент сам ретраит.
2. **Итерация «докажи»** в каждый план и каждого агента (анти-код-ради-кода).
3. **4-вопросная валидация фаз** + **pre-mortem** для крупных фич в шаблон `plans/`.
4. **Скилл `idea-validation`** (мастерство) с триггером «испытать идею» — гейт для новых фич перед планом.
5. **Защитный хук на `.env`** — запрет чтения ключей агентом + проверка устанавливаемых скиллов.

### Автономные агенты (когда дойдёшь до автоматизации)
6. **SEO-отдел из автономных агентов** — агенты генерят SEO-статьи под поисковую выдачу ниши → бесплатный органический трафик.
7. **Робот-звонилка по лидам** — автозвонок сразу после заявки/брошенной корзины с переводом на оператора (у Миллера дал конверсию ×2).
8. **Ежемесячный агент x2** — читает `.business/economics`, `goals`, метрики → «что дало 80% результата / что убрать».
9. **Мониторинг + алерты** — парсер рекламных кабинетов с алертами о сливе бюджета.

### Прочее
10. **Файл «методология дизайна»** + ссылка в CLAUDE.md — единый фирменный стиль всех страниц.
11. **Qwen Coder** — дешёвый второй пилот (~$30 = лимит Claude за $100); полезен для второго прохода security-review; без VPN.
12. **CRM с нуля вместо платной** — на базе опенсорса, если готовая «под вопросом».
13. **Обезличивание перс. данных** при ИИ-обработке (без телефона/email, привязка к ID после).
14. **Самообучение через ретроспективы** — агент перед новой фичей читает `.business/история/`.
